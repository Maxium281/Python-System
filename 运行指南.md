# Python面试系统 - 运行指南

## 系统要求
- Python 3.7 或更高版本
- pip（Python包管理器）

## 快速开始

### 1. 安装依赖

在项目根目录下打开终端/命令行，运行：

```bash
pip install -r requirements.txt
```

或者如果使用Python 3，可能需要使用：

```bash
pip3 install -r requirements.txt
```

### 2. 检查数据库

系统使用SQLite数据库，数据库文件位于 `database/interview.db`。

如果数据库不存在，系统会在首次运行时自动创建（如果配置了自动初始化）。

### 3. 运行应用

在项目根目录下运行：

```bash
python run.py
```

或者：

```bash
python3 run.py
```

### 4. 访问应用

应用启动后，在浏览器中访问：

```
http://127.0.0.1:5000
```

或

```
http://localhost:5000
```

## 主要功能页面

- **首页**: `http://127.0.0.1:5000/`
- **题目分类**: `http://127.0.0.1:5000/question/category`
- **模拟面试**: `http://127.0.0.1:5000/exam/mock`
- **学习进度**: `http://127.0.0.1:5000/progress/`

## 配置说明

应用配置在 `app/core/config.py` 中，可以通过环境变量修改：

- `DEBUG`: 调试模式（默认：False）
- `HOST`: 服务器地址（默认：127.0.0.1）
- `PORT`: 端口号（默认：5000）
- `DEFAULT_QUESTION_COUNT`: 默认题目数量（默认：20）
- `DEFAULT_TIME_LIMIT`: 默认考试时长（秒，默认：1800，即30分钟）

## 常见问题

### 1. 端口被占用

如果5000端口被占用，可以修改 `run.py` 中的端口号：

```python
app.run(host='127.0.0.1', port=8080, debug=False, threaded=True)
```

### 2. 数据库错误

如果遇到数据库相关错误，可以：
- 检查 `database/` 目录是否存在
- 检查数据库文件权限
- 查看 `logs/interview_system.log` 日志文件

### 3. 依赖安装失败

如果安装依赖时遇到问题：
- 确保使用正确的Python版本
- 尝试使用 `pip install --upgrade pip` 升级pip
- 检查网络连接

## 开发模式

如果需要启用调试模式，修改 `run.py`：

```python
app.run(host='127.0.0.1', port=5000, debug=True, threaded=True)
```

## 停止应用

在运行应用的终端中按 `Ctrl + C` 停止服务器。

## 日志文件

应用日志保存在 `logs/` 目录下：
- `interview_system.log`: 主日志文件
- `interview_system_error.log`: 错误日志
